Genericity
  • A model is the definition of the formatted data on which the program works. This is where the attribute(s) and the relation(s), that describe it, are defined
    (ex: a geometric shape, described by its shape, color and size)
  • An object is an instantiation of the model
    (ex: a square, red of size 8)
  • An example (or counter-example) is a set of objects. Each example is independant of the others

\( \implies \) generic architecture : works with any model and exemples

From a semantic point of view : logical consistency of input data is not checked, whether a representation exist or not
(ex: possible physical representation)
